# Onboard AITeamOps（自举：教一个新模型学会并永久记住这套规则）

把 AITeamOps 接入「当前这个 AI 工具」，让它的规则与 Skill 在**该用户的所有项目里持久生效**——不是读一遍就忘，而是写进宿主工具每次会话都会加载的文件。

## 最重要的认知边界（先讲清，别给用户错觉）

- **模型没有「永久记忆」，持久化靠宿主工具的文件机制。** 读一个 GitHub 仓库只是临时上下文，会话结束即忘。**只有把治理块写进该工具每次加载的文件，才算"永久记住"。**
- **各工具的记忆文件不同**：Claude Code → `~/.claude/CLAUDE.md`；Codex → 全局 `AGENTS.md`（如 `~/.codex/AGENTS.md`）；Cursor / 其它 → 其各自的全局规则文件。拿不准就问用户，不要假设。
- **"让你的模型学习仓库" ≠ 自动永久记住。** 必须经过本流程的"落盘 + 自检"两步，且落盘改的是全局配置 → 要用户确认。
- 本 Skill 能不能在别的模型上奏效，取决于那个模型是否愿意按 ONBOARDING.md / 本流程执行；它是**给模型的明确指令**，不是魔法。

## 红线

1. **写全局/记忆文件前必须用户确认。** 影响该用户所有项目，不可悄悄改。
2. **只写精炼「治理块」，不抄全文。** 各 Skill 完整流程留在仓库，治理块只放规则摘要 + 触发清单 + 仓库路径。
3. **增量追加，不覆盖用户已有规则。** 在目标文件加一段独立小节，绝不重写或删改用户原有内容。
4. **先探测，再落盘。** 先确认是什么工具、目标文件在哪、是否已接入过（避免重复写）。

## 执行流程

### 第 1 步：读懂框架
按顺序读：`README.md` → `AGENTS.md` → `ROADMAP.md` → `.agents/skills/*/SKILL.md`。重点记住：三层记忆、读写闭环、各 Skill 的触发场景与红线。

### 第 2 步：判定当前工具与它的「永久记忆文件」
```text
Claude Code → ~/.claude/CLAUDE.md（全局，所有项目加载）
Codex       → 全局 AGENTS.md（如 ~/.codex/AGENTS.md）
Cursor      → 其全局/用户规则文件（以其文档为准）
其它工具    → 该工具会每次加载的全局指令文件
```
拿不准当前是什么工具、文件在哪 → **问用户**，不要猜。

### 第 3 步：安装 Skills 到该工具可发现的位置（可选但推荐）
- Windows：可用 `scripts/install-windows.ps1`（把 `.agents/skills/*` 复制到 `~/.agents/skills/`）。
- 或不复制，只在治理块里写明仓库路径，命中场景时按需读对应 `SKILL.md`（省 token）。

### 第 4 步：拟「治理块」→ 用户确认 → 增量写入
把下面的治理块按当前环境填好（仓库实际路径、用户在用的工具），**原文给用户看，确认后**追加到第 2 步定位的文件，不动其它内容。

### 第 5 步：自检验证（证明真的"记住"了）
落盘后做一次自检并报告：
- 复述三层记忆分层规则（项目事实/跨项目经验/一次性失误各去哪）。
- 复述「首次浏览代码前先 pull」「提交需 push 确认」两条。
- 说出至少 3 个 Skill 的触发场景。
- 告诉用户：写进了哪个文件、加了哪一段、如何回退（删除该段即可）。

## 治理块模板（写进宿主工具的记忆文件）

```markdown
## AITeamOps 团队 AI 协作规范（对所有项目生效）
框架仓库：<填本仓库路径或 URL>。完整流程在 <仓库>/.agents/skills/<name>/SKILL.md，命中场景按需读。

### 三层记忆分层
- 项目专属事实 → 项目内 docs/ai-memory/（ai-handoff-doc-update）
- 跨项目通用经验 → 本文件下方「跨项目经验」区（cross-project-experience，改前确认）
- 一次性操作失误 → 不记（除非提炼出可预防的通用模式）

### 开工与提交
- 首次浏览/探索任一项目代码前，先 git pull --ff-only 同步远程（无仓库/无 remote 跳过；
  dirty 或冲突停下交人，不强拉）。对应 project-context-sync。
- 提交用 git-commit-guard：先 git add -A 再扫敏感信息（含未跟踪文件）→ 构建/测试 →
  全绿才 commit；push 需显式确认；默认提交当前分支。

### 可用 Skill（命中即用，全文见仓库）
project-context-sync 开工同步｜ai-handoff-doc-update 任务后沉淀项目记忆｜
cross-project-experience 跨项目经验沉淀到本文件｜multi-tool-entrypoint-sync 多工具入口统一｜
git-commit-guard 安全提交｜mysql-readonly-probe-via-java 只读探测｜team-ai-workspace-bootstrap 新项目骨架

### 跨项目经验（自动累积，由 cross-project-experience 维护；改前确认）
- （示例）合并若没先 pull、或对冲突没看就合，必须三方核对（合并结果 vs 本地父 vs 远程父），
  确认本地已修复项没被远程改动悄悄回退。
```

## 与其它 Skill 的关系

- 落盘后，跨项目经验的**持续累积**由 `cross-project-experience` 接手（本 Skill 只负责首次接入）。
- 多工具一致用 `multi-tool-entrypoint-sync`（项目级入口）；本 Skill 管的是**全局级**接入。

## 常见坑

### 坑 1：给用户"读一遍就永久会了"的错觉
不写进宿主文件就不持久。必须落盘 + 自检，并如实说明这是一次安装动作。

### 坑 2：把整套 SKILL 全文抄进全局文件
全局文件每次会话都加载，越短越好。只写治理块 + 仓库路径，全文按需读。

### 坑 3：覆盖用户已有的全局规则
只增量加一段独立小节。先读现有内容，避免重复接入。

### 坑 4：假设当前是什么工具
Claude/Codex/Cursor 的记忆文件各不相同。先探测或问用户，再落盘。
