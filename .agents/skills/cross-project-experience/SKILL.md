# Cross-Project Experience（跨项目经验沉淀 / 全局累积）

把「对任何项目都适用的可复用经验」沉淀到**模型的全局指令文件**，让 AI 跨项目地越用越强——这是一个微型的、不断累积经验的过程。它是 `ai-handoff-doc-update`（项目记忆，写侧）的「上一层」：那个管单个项目的事实，本 Skill 管所有项目通用的经验。

## 三层记忆：先判断这条经验属于哪一层

```text
经验产生时先判层：

① 只对「这个项目」成立的事实        例：某字段 keyProject=0 表示"是"（非标准约定）
     → docs/ai-memory/             用 ai-handoff-doc-update，随项目 git 共享给团队

② 对「任何项目」都成立的可复用经验  例：无看合并必须三方核对；改前先 pull
     ├─ 行为/原则类  → 全局指令文件（本 Skill）—— 影响所有项目，改前必须确认
     └─ 能固化成流程 → 提议新增/更新一个 Skill（写进 AITeamOps）

③ 一次性失误、无复用价值            例："这次忘了 pull"本身
     → 不记。但若它暴露了可预防的通用模式，把"模式"提炼进 ②
```

**只有第 ② 层归本 Skill。** 第 ① 层交给 `ai-handoff-doc-update`；第 ③ 层不记。

## 认知边界

- 全局指令文件影响你**每一个**项目、每一次会话。往里写 = 改变所有项目的默认行为，慎之又慎。
- 「通用」要保守判定：能举出 **≥2 个不同项目**都适用，才算通用；只在当前项目成立的是项目事实，不往全局写。
- 不是所有通用经验都塞全局文件——**能固化成流程的，优先提议成一个 Skill**，全局文件只放精炼的「行为/原则」。

## 红线

1. **写全局指令文件前必须用户确认。** 影响所有项目，不可悄悄改。
2. **不记一次性操作失误本身**（如"这次忘了 pull"）；只在它暴露可预防的通用模式时，把「模式」写成一条经验。
3. **不把项目专属事实写进全局**（会污染所有项目）。项目事实 → `ai-handoff-doc-update` → `docs/ai-memory`。
4. **增量追加 + 去重。** 同一条经验只留一份，相近的合并，不堆重复、不自相矛盾。
5. **保持精炼。** 全局文件每次会话都加载，越短越省 token。

## 判定清单：这条经验该不该进全局？

```text
□ 在 ≥2 个不同项目都会用上？        否 → 不进全局（可能是项目事实，走 ①）
□ 从代码/git 看不出、却容易反复踩？  是 → 值得记
□ 能固化成一套步骤？                能 → 优先提议成 Skill，而非塞进全局文件
□ 是一次性失误？                    是 → 不记（除非提炼出可预防模式）
```

## 执行流程

### 第 1 步：判层
对照三层模型，确认是第 ② 层（通用）。不是 → 交回 `ai-handoff-doc-update`（①）或丢弃（③）。

### 第 2 步：定目标文件（按当前所用工具）
- Claude Code → `~/.claude/CLAUDE.md`（或它 `@import` 的全局经验文件）
- Codex → 全局 `AGENTS.md`（如 `~/.codex/AGENTS.md`）
- 其它工具 → 该工具的全局指令文件

以用户**实际所用工具**为准；拿不准先问，不要假设。

### 第 3 步：写之前先去重
读目标文件已有的「跨项目经验」区，看这条是否已存在/相近。已有 → 精准订正那一条，不新增重复。

### 第 4 步：拟稿 + 用户确认（红线 1）
把要写入的精炼条目原文给用户看，确认后再落盘。

### 第 5 步：增量追加
在目标文件的「跨项目经验」区追加一条，用下面的格式。**不动用户其它规则内容。**

## 经验条目格式

```text
- <一句话经验>。场景：<什么时候适用>。为什么：<不记会怎样 / 根因>。
```

示例（来自一次「没先 pull 就合并、对冲突没看就合」的真实场景）：

```text
- 合并若没先 pull、或对冲突没看就合，必须做三方核对（合并结果 vs 本地父 vs 远程父），
  确认本地已修复项没被远程改动悄悄回退。场景：任何无看 / 仓促合并后。为什么：远程的
  "修复"可能与本项目既定约定相反，机械合并会把回退吞进来，让改好的问题复发。
```

## 与其它 Skill 的关系

- `ai-handoff-doc-update`：项目事实（第 ① 层）。本 Skill 是它的**全局对偶**（第 ② 层）。
- 经验能固化成流程 → 按 `CONTRIBUTING.md` 在 AITeamOps **新增一个 Skill**（第 ② 层的「流程」出口）。
- `project-context-sync`：开工时加载全局经验，也属于上下文对齐的一部分。

## 常见坑

### 坑 1：把项目专属事实写进全局
会污染所有项目。判层要严：只在当前项目成立的，永远走 `docs/ai-memory`。

### 坑 2：把一次性失误当经验记
"这次忘了 pull"是失误不是经验。只记从它提炼出的**可预防通用模式**。

### 坑 3：不去重就追加
全局文件会膨胀、甚至自相矛盾。先读已有内容，相近的合并订正。

### 坑 4：不经确认改全局
影响面失控。改全局指令文件永远先把条目给用户确认（红线 1）。
